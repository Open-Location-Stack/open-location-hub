# Engineering Documentation

Use the documents in this directory for project development workflow, governance, and validation expectations. Use GitHub issues for current implementation tracking and follow-up work.

- `analytics-project-plan.md`: draft plan for a separate downstream analytics and Grafana project built on hub event APIs
- `federation-plan.md`: architecture and delivery plan for hub-to-hub federation across on-prem, regional cloud, and aggregate cloud tiers
- `openapi-governance.md`: contract-first API rules and OMLOX alignment guidance
- `testing.md`: repository validation workflow and test harness guidance
